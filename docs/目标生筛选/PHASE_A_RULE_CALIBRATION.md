## 1. 目标

统一“目标生筛选”第一期规则口径，确保产品、后端、前端、测试、教务对结果解释一致，避免联调与验收争议。

---

## 2. 冻结范围

第一期仅支持“单条件 + 时序量词”：

1. 指标：total_score_rank_in_grade（总分年级排名）
2. 比较：lte（小于等于）
3. 阈值：前N名（N为正整数）
4. 量词：all / at_least
5. 缺考策略：strict_fail / ignore_absent

非范围项：

1. 多条件AND/OR
2. 学科维度规则
3. 连续K次、最近K次
4. 规则保存与共享

---

## 3. 关键术语与定义

1. 目标考试集合 E：按 grade_level + exam_scope 解析出的考试集合。
2. 候选学生集合 S：默认该年级在读学生（status=在读）。
3. 命中（hit）：某学生在某考试满足 rank <= threshold。
4. 缺考（miss）：该学生在目标考试中不存在可判定的总分年级排名。

---

## 4. 规则口径冻结结论

## 4.1 exam_scope 口径

支持三种类型：

1. all_in_grade：该年级全部考试。
2. selected_exam_ids：指定考试ID列表。
3. date_range：起止日期区间内的该年级考试。

冻结结论：

1. exam_scope 解析后若 E 为空，接口返回400。
2. selected_exam_ids 不属于 grade_level 的考试将被过滤；过滤后为空仍返回400。

## 4.2 quantifier 口径

1. all：每次都满足。
2. at_least：至少K次满足。

冻结结论：

1. quantifier=at_least 时 k 必填且 > 0。
2. strict_fail 场景下，k > |E| 判定为非法参数并返回400。

## 4.3 absent_policy 口径

1. strict_fail：缺考视为不满足。
2. ignore_absent：缺考不记失败，不参与分母。

冻结结论：

1. quantifier=all + ignore_absent 时，要求参与场次 >= 1 且参与场次全部命中。
2. quantifier=at_least + ignore_absent 时，仅要求命中次数 >= k。

## 4.4 排名与并列口径

1. 排名直接使用现有写入字段 total_score_rank_in_grade。
2. 并列排名沿用当前系统并列逻辑，不二次重排。

---

## 5. 边界口径冻结

1. threshold <= 0：返回400。
2. quantifier 非法：返回400。
3. operator 非 lte：返回400（第一期仅支持 lte）。
4. grade_level 缺失或非法：返回400。
5. 无命中学生：返回 success=true，students=[]。

---

## 6. 争议点与最终决议

1. 争议：候选学生是否包含非在读状态。
- 决议：第一期仅在读学生；第二期可配置扩展。

2. 争议：ignore_absent + all 是否允许0参与。
- 决议：不允许；至少参与1场，避免“全缺考却被命中”。

3. 争议：指定考试跨年级是否可查。
- 决议：按grade_level严格过滤，避免口径混乱。

---

## 7. 评审检查清单

1. 口径是否可被测试脚本直接复现。
2. 口径是否可被页面文案准确表达。
3. 口径是否在接口字段上可完整表达。
4. 口径是否覆盖缺考与边界场景。
