# PRD：个人成绩分析导出

## 1. 文档信息
- 文档名称：个人成绩分析导出 PRD
- 所属模块：成绩分析 > 个人成绩分析
- 版本：v1.0
- 日期：2026-04-07
- 状态：待开发

## 2. 背景与问题
当前系统已支持“个人成绩分析”页面展示趋势图、排名、科目明细，但缺少可归档、可流转、可打印的标准化导出能力。

业务侧需要类似“学生学业水平跟踪档案与分析”的报告文件，用于：
- 家校沟通（学生个体报告）
- 班主任与教研组复盘
- 阶段性质量跟踪归档

## 3. 目标
实现“个人成绩分析导出（Excel）”能力，满足以下目标：
1. 可从个人成绩分析页一键导出。
2. 导出内容结构清晰，包含总览、趋势、科目明细与排名变化。
3. 科目顺序严格遵循 SUBJECT_CHOICES。
4. 首版以“可用、稳定、可维护”为优先，后续再增强看板化样式。

## 4. 非目标
以下内容不纳入本期：
1. 完全复刻复杂商业模板（例如切片器、复杂透视联动、VBA）。
2. PDF 版本导出。
3. 多学生批量生成个人报告包（zip）。

## 5. 用户与场景
### 5.1 目标用户
- 管理员
- 年级管理员
- 教师（有成绩分析权限）

### 5.2 核心场景
1. 教师进入个人成绩分析详情页，选择学生后点击“导出个人分析报告”。
2. 系统生成 Excel 并下载到本地。
3. 教师用于线下沟通、归档或进一步汇报。

## 6. 功能范围
### 6.1 前端
1. 在个人成绩分析详情页新增“导出报告”按钮。
2. 点击后调用后端导出接口，触发文件下载。
3. 导出中显示按钮 loading 状态，失败给出明确错误提示。

### 6.2 后端
1. 新增个人分析导出接口（GET）。
2. 基于现有 student_analysis_data 的数据口径生成 Excel。
3. 生成多工作表报告并设置基础样式。
4. 接口鉴权与现有成绩分析一致。

## 7. 报告结构（首版）
导出文件名建议：
- 个人成绩分析报告_{学生姓名}_{学号}_{YYYYMMDD_HHmm}.xlsx

工作表设计：
1. 总览
- 建议分区展示（从上到下）：
  - A. 学生信息区：姓名、学号、年级、班级、分析时间、分析覆盖考试数。
  - B. 总分概览区：总分均分、最高总分（含考试名/日期）、最低总分（含考试名/日期）、总分波动区间（最高-最低）。
  - C. 排名概览区：最佳年级排名（含考试名/日期）、最差年级排名（含考试名/日期）、相对首场考试排名变化、期初到期末排名净变化。
  - D. 最新考试快照区：最新总分、最新班级/年级排名、较上次总分变化、较上次年级排名变化。
  - E. 最新科目快照区：各科“得分+班排+级排”（按 SUBJECT_CHOICES 顺序输出）。
- 字段规则：
  - “较上次年级排名变化”= 上次年级排名 - 本次年级排名（正数=提升，负数=下降）。
  - “相对首场考试排名变化”= 首场年级排名 - 最新年级排名（正数=较首场提升）。
  - “期初到期末排名净变化”默认与“相对首场考试排名变化”同口径。
  - 若仅有1场考试，则所有“较上次”字段填“-”。
  - 若存在缺考或缺失排名，不影响其他可计算字段；不可计算字段填“-”。
- 排版草图（建议，Excel 竖版打印友好）：

| 区域 | 单元格建议 | 内容 | 样式建议 |
|---|---|---|---|
| 标题区 | A1:L1（合并） | 个人成绩分析报告（姓名+学号） | 16号加粗、居中、浅灰底 |
| 元信息区 | A2:F4 | 学生信息（姓名、学号、年级、班级、分析时间、覆盖考试数） | 左对齐，标签列加粗 |
| B.总分概览区 | G2:L4 | 均分、最高分、最低分、波动区间 | 数值右对齐，关键值加粗 |
| C.排名概览区 | A5:L7 | 最佳/最差年级排名、相对首场排名变化、期初到期末净变化 | “变化”字段可用条件色 |
| D.最新考试快照区 | A8:L10 | 最新总分、最新班级/年级排名、较上次总分变化、较上次排名变化 | 与趋势页同口径配色 |
| E.最新科目快照区 | A11:L20（按科目数扩展） | 各科得分/班排/级排（按 SUBJECT_CHOICES） | 表头深色底白字，隔行底纹 |
| 备注区 | A21:L22（可选） | 口径说明：排名变化正数=提升、负数=下降 | 10号字、灰色文本 |

布局示意（逻辑分块）：

| 行区间 | 左侧（A-F） | 右侧（G-L） |
|---|---|---|
| 1 | 标题 | 标题 |
| 2-4 | 学生信息区 | 总分概览区 |
| 5-7 | 排名概览区（跨整行） | 排名概览区（跨整行） |
| 8-10 | 最新考试快照区（跨整行） | 最新考试快照区（跨整行） |
| 11-20 | 最新科目快照区（跨整行） | 最新科目快照区（跨整行） |

实现提示：
- 列宽建议：A列14、B-F列12、G-L列13；长文本列（考试名）可设18-22。
- 冻结窗格建议：A11（冻结标题与概览信息，便于滚动查看科目区）。
- 打印建议：纸张A4、纵向、1页宽自适应、上下页边距适中。

2. 总分趋势
- 每次考试：考试名称、日期、总分、班级排名、年级排名
- 建议补充字段（用于趋势判读）：
  - 较上次总分变化（本次总分 - 上次总分）
  - 较上次年级排名变化（上次年级排名 - 本次年级排名，正数=提升）
  - 滚动均分（近3次，不足3次按已有次数计算）
  - 相对首场考试的排名变化（首场年级排名 - 本次年级排名，正数=提升）
- 展示规则：
  - 表格按日期升序，确保“上次”计算稳定。
  - 首场考试无“上次”可比时，变化列填“-”。
  - 若某场总分或排名缺失，则该场相关变化列填“-”，不参与滚动均分计算。
  - 建议在工作表顶部增加“趋势摘要”区：最高分、最低分、最大单次进步、最大单次回落、期初到期末排名净变化。
- 图表（本期纳入范围）：
  - 图表1：总分趋势折线图（X轴=考试日期，Y轴=总分）。
  - 图表2：年级排名趋势折线图（X轴=考试日期，Y轴=年级排名，Y轴反向）。
  - 图表3：总分3次滚动均分折线（与图表1同图叠加，颜色区分）。
  - 图表放置建议：位于“总分趋势”工作表数据表下方，默认宽屏打印可完整显示。
  - 数据点不足规则：
    - 有效考试场次 < 2：不生成图表，仅保留表格与提示“数据不足，未生成趋势图”。
    - 仅部分字段缺失：图表按可用数据点绘制，缺失点断开。

3. 科目明细（主表，宽表模式）
- 维度：每场考试一行（不是“考试 x 科目”长表）
- 固定前置列：日期、考试名、总分、总分级排
- 科目横向展开列（按 SUBJECT_CHOICES 顺序）：
  - 语文得分、语文班排、语文级排
  - 数学得分、数学班排、数学级排
  - 英语得分、英语班排、英语级排
  - ……以此类推直到该次考试包含的全部科目
- 缺失科目或缺考统一填“-”
- 首版不强制输出“科目满分/得分率”列（可作为二期增强）

4. 科目趋势
- 每场考试一行，展示“总分级排 + 各科级排 + 相对上次级排变化”
- 变化值定义：上次级排 - 本次级排（正数=提升，负数=下降）
- 图表（本期纳入范围）：
  - 图表4：最新一次考试“各科级排变化”柱状图（X轴=科目，Y轴=级排变化值）。
  - 正数（提升）使用绿色系，负数（下降）使用红色系，0 使用中性色。
  - 图表放置建议：位于“科目趋势”工作表数据表下方。
  - 数据点不足规则：若最新考试无可比较的“上次级排”数据，则不生成该图并显示“暂无可比较科目趋势”。

### 7.2 图表实现约束（本期）
1. 图表由后端导出时自动生成，无需前端二次处理。
2. 图表数据源必须来自同一工作表内可见数据区域，避免引用隐藏计算区。
3. 图表颜色遵循模板主色系：
- 主色：#00897B（趋势主线）
- 提升：绿色系
- 下降：红色系
4. 图表标题命名固定：
- 总分趋势（含滚动均分）
- 年级排名趋势
- 各科级排变化（最新考试）
5. 若图表生成失败，不影响文件导出：
- 文件照常下载，且在对应工作表顶部写入“图表生成失败，请联系管理员”。

## 8. 数据口径与排序规则
1. 数据来源与个人分析页一致，保证导出与页面一致性。
2. 科目排序：科目横向列严格按 SUBJECT_CHOICES 顺序；不在 SUBJECT_CHOICES 的科目追加到末尾。
3. 日期排序：按考试日期升序。
4. 缺失值：统一输出“-”。
5. 条件格式：
- 仅对“排名变化”列应用条件格式。
- 变化值 > 0（提升）：绿色填充/绿色字体。
- 变化值 < 0（下降）：红色填充/红色字体。
- 变化值 = 0：默认样式或中性色。

## 9. 权限与安全
1. 仅登录用户可访问。
2. 写权限用户可导出；如与现有分析权限一致则复用同一权限策略。
3. 接口需校验 student_id 合法性与可见范围。
4. 响应头设置为附件下载，避免浏览器内联渲染。

## 10. 交互与错误处理
### 10.1 前端交互
1. 按钮文案：导出个人报告。
2. 导出中：按钮禁用，文案“导出中...”。
3. 成功：浏览器下载文件。
4. 失败：toast/alert 提示具体错误。

### 10.2 异常处理
1. 学生不存在：返回 404 + 错误信息。
2. 无考试数据：返回 400 +“该学生暂无可导出分析数据”。
3. 服务异常：返回 500 + 通用错误信息。

## 11. 接口草案
- 路径：/api/scores/student-analysis-report-export
- 方法：GET
- 参数：
  - student_id: number（必填）
  - exam_ids: string（可选，逗号分隔）
- 返回：application/vnd.openxmlformats-officedocument.spreadsheetml.sheet

响应头：
- Content-Disposition: attachment; filename="个人成绩分析报告_xxx.xlsx"

## 12. 验收标准
1. 个人成绩分析详情页可见导出按钮并可触发下载。
2. 导出文件包含本 PRD 定义的 4 个工作表。
3. 表格中的科目顺序符合 SUBJECT_CHOICES。
4. 导出核心数值与页面一致（抽样核对 3 名学生通过）。
5. 无数据和异常场景提示符合预期。
6. “排名变化”列条件格式生效（提升为绿色，下降为红色）。
7. 本期定义的 4 张图表按规则生成；数据不足场景按 PRD 规则降级显示。

## 14. 风险与对策
1. 风险：科目顺序在不同接口中不一致。
- 对策：统一在后端导出层按 SUBJECT_CHOICES 重排。

2. 风险：学生考试场次较多时导出耗时。
- 对策：首版同步导出，后续可增加异步导出任务队列。

3. 风险：模板样式需求持续升级。
- 对策：先交付标准版结构，第二期增加看板化模板。
