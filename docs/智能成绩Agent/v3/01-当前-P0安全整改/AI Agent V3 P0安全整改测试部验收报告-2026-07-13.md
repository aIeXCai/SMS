## 一、测试范围

本次回归覆盖 AI Agent V3 P0 安全整改四类场景：

1. 权限：`admin` / `staff` 全校可查；`grade_manager` 仅 `managed_grade`；`subject_teacher` 仅 `teaching_classes`；越权统一提示，不暴露目标是否存在。
2. 脱敏：发送给 MiniMax 的 Tool observation 不含学号、手机号、邮箱、原始请求体、密钥；学生姓名需脱敏。
3. 数字可信：分数、排名、人数、变化量等关键数字必须来自后端结构化 Tool 结果；LLM 改写或新增数字时应拦截。
4. 降级：简单白名单查询可降级；趋势、加权、群体对比、多轮追问、权限失败、数字校验失败不得回退 V1。

## 四、边界情况

1. 敏感 key 当前只做精确小写匹配，未覆盖 `raw_request_body`、`request_body`、`secret_key`、`Authorization`、`token`、`apiKey` 等常见变体。
2. `numeric_guard.extract_numeric_facts` 只采集 `int` / `float`，不采集字符串中的可信数字，例如百分比字符串。
3. 测试配置关闭 V3，API 层真实 V3 路由需要单独用 mock LLM 覆盖。

## 五、Bug 列表

### Bug 1：Tool observation 最小化未过滤原始请求体和密钥类字段

| 字段 | 内容 |
|---|---|
| 严重程度 | P0 |
| 复现步骤 | 调用 `minimize_tool_result("get_scores", {"raw_request_body": {...}, "secret_key": "sk", "Authorization": "Bearer token"})` |
| 实际结果 | `raw_request_body`、`secret_key`、`Authorization` 原样保留 |
| 预期结果 | 发送给 MiniMax 的 observation 不包含原始请求体、密钥、token、Authorization 等敏感信息 |
| 可能原因 | `minimizer.py` 的 `SENSITIVE_KEYS` 覆盖不足且大小写敏感 |
| 建议修复方向 | key 统一 lower / snake 归一后匹配，补充 `raw_request_body`、`request_body`、`body`、`secret`、`secret_key`、`token`、`authorization`、`apiKey` 等 |
| 是否阻塞发布 | 是 |

### Bug 2：字符串百分比数字不进入可信数字事实，可能误拦截合法跨群体结果

| 字段 | 内容 |
|---|---|
| 严重程度 | P1 |
| 复现步骤 | Tool 结构化结果包含 `{"ratio": "103.9%"}`，LLM 总结中复述 `103.9%` |
| 实际结果 | `103.9` 不进入 numeric facts，LLM 文本校验失败 |
| 预期结果 | 后端结构化 Tool 结果中的百分比 / 数字字符串也应可追溯 |
| 可能原因 | `extract_numeric_facts` 只采集数值类型，不解析字符串中的数字 |
| 建议修复方向 | 对明确结构化字段中的数字字符串提取 numeric facts，如 `ratio`、`exam_a_weight`、`exam_b_weight` 等 |
| 是否阻塞发布 | 否，但影响跨群体对比可用性 |
